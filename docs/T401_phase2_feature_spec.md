> 관련 태스크: T401 — Phase 2 착수 전 상세 기능 명세 확정 (Phase 1의 T101과 동일한 역할)

## 개요

Phase 2는 **디지털 다꾸 (콘텐츠 확장)** 단계로, Phase 1에서 수집·등록한 도감 아이템을 "스티커"로 재활용해 달력 위에 자유롭게 배치하는 다이어리 꾸미기 경험을 제공한다. 서버 없이 로컬 저장소만 사용하는 것은 Phase 1과 동일하다.

### 기획안 원문 (근거)

> - **스티커 자동 연동**: 도감에 수집 및 등록된 일상의 조각들이 다꾸 스페이스에서 곧바로 불러올 수 있는 나만의 커스텀 스티커로 변환
> - **다꾸 스페이스**: 빈 캔버스 혹은 달력 폼 위에 내 도감 스티커들을 자유롭게 배치(드래그앤드롭, 확대/축소, 회전)하여 꾸미는 기능
> - **달력(캘린더) 뷰**: 일기를 매일 쓰는 부담을 줄이기 위해, 달력 날짜 칸에 수집한 엽서 조각 하나만 툭 붙여도 감성적인 일간 기록이 완성되는 직관적 캘린더 인터페이스

### 핵심 기능 요약
1. **캘린더 뷰** - 월별 달력, 날짜별 다꾸 미리보기 썸네일, 날짜 탭으로 그날의 다꾸 목록 진입 (하루 여러 장 지원)
2. **날짜별 다꾸 스페이스** - 도감 아이템을 스티커로 불러와 드래그/확대/회전 배치 (T203에서 검증한 제스처 로직 재사용)
3. **다꾸 결과물 저장** - 배치가 바뀔 때마다 자동 저장되며, 별도의 명시적 "저장" 버튼으로도 확정 가능. 완성 화면은 PNG로 내보내기/공유

### Phase 2 MVP 범위가 아닌 것
- **자유 캔버스(날짜 미연결) 다꾸**: 기획안은 "빈 캔버스 혹은 달력 폼"이라 언급하지만, MVP는 달력 연동에 집중한다. T203에서 만든 스티커 캔버스(이모지 데모)는 제스처 기술 검증용으로 남겨두고, 실제 도감 아이템을 쓰는 날짜 미연결 자유 캔버스는 Phase 2 이후로 보류
- **친구 공유·소셜 기능**: Phase 3 범위 (`app_planning_pong_dang.pdf` [Phase 3] 참고)

## Feature 7: 캘린더 뷰 (Calendar View)

| 항목 | 내용 |
|------|------|
| **기능 ID** | F7-CALENDAR |
| **목적** | 날짜 기반으로 다꾸 기록을 탐색하고, 새 기록의 진입점을 제공 |

### 세부 기능

| ID | 기능명 | 설명 | 입력 | 출력 |
|----|--------|------|------|------|
| F7.1 | 월별 캘린더 그리드 | 현재 월의 날짜를 그리드로 표시, 좌우 스와이프/화살표로 월 이동 | 사용자 액션 | 월별 날짜 그리드 |
| F7.2 | 날짜별 미리보기 | 다꾸 기록이 있는 날짜 칸에 가장 최근 다꾸의 대표 스티커(첫 배치 아이템) 썸네일 표시. 하루에 여러 장이 있으면 개수 배지(예: "+2") 함께 표시, 없으면 빈 칸 | 해당 날짜의 다꾸 데이터 | 썸네일 + 개수 배지 |
| F7.3 | 오늘 표시 | 오늘 날짜를 시각적으로 강조, "오늘로 이동" 버튼 제공 | 시스템 날짜 | 강조된 오늘 셀 |
| F7.4 | 날짜 진입 | 날짜 칸 탭 시 그날의 다꾸 목록 바텀시트를 표시(썸네일 리스트, 하루 여러 장 지원) — 항목 탭하면 해당 다꾸 스페이스로, "+ 새 다꾸 추가"로 그날짜의 새 다꾸 스페이스 생성. 기록이 하나도 없으면 시트 없이 바로 새 다꾸 스페이스로 진입. **단, "새 다꾸 스페이스로 진입"은 화면만 여는 것이지 저장이 아님** — 실제로 저장(F8.6) 전까지는 `DecoEntry`가 생기지 않아, 열어보기만 하고 나가면 캘린더에 아무 흔적도 남지 않음 | 날짜 탭 | 다꾸 목록 시트 또는 다꾸 스페이스 화면 |
| F7.5 | 날짜 제한 없음 | 과거/오늘/미래 모든 날짜에 동일하게 접근·조회·편집 가능 — 잠금이나 "오늘 이후만" 같은 제약 없음. 지난 날짜를 나중에 회고하며 꾸미거나, 예정된 날짜를 미리 꾸며두는 것 모두 허용 | 날짜 탭 | 제약 없이 다꾸 스페이스 진입 |

### UI 레이아웃
- 상단: 연/월 표시 + 이전/다음 월 이동 버튼 + 오늘로 이동
- 본문: 7열(요일) x N행 그리드, 각 셀은 날짜 숫자 + (있으면) 스티커 썸네일 + 개수 배지
- 홈 화면 하단 네비게이션에 "도감 / 폴더" 옆 세 번째 탭 "달력"으로 진입 (T303에서 만든 `NavigationBar` 확장)

## Feature 8: 다꾸 스페이스 (Deco Space)

| 항목 | 내용 |
|------|------|
| **기능 ID** | F8-DECO |
| **목적** | 특정 날짜에 도감 아이템(스티커)을 자유롭게 배치해 일간 기록을 완성 |

### 세부 기능

| ID | 기능명 | 설명 | 입력 | 출력 |
|----|--------|------|------|------|
| F8.1 | 스티커 불러오기 | 도감 아이템 목록(그리드)에서 선택해 캔버스에 스티커로 추가 | 아이템 선택 | 캔버스에 배치된 스티커 |
| F8.2 | 배치 제어 | 스티커/텍스트 공통 — 드래그(이동)/핀치(확대·축소)/회전. T203의 `onScale*` 로직 재사용 | 제스처 입력 | 갱신된 위치/크기/각도 |
| F8.3 | 겹침 순서 | 스티커/텍스트 탭 시 맨 앞으로 이동(z-index 갱신) | 탭 | 갱신된 겹침 순서 |
| F8.4 | 요소 삭제 | 선택한 스티커 또는 텍스트를 캔버스에서 제거 (도감 원본 아이템은 삭제되지 않음) | 삭제 액션 | 배치 제거 |
| F8.5 | 자동 저장 | 배치가 바뀔 때마다 메모리 상 상태는 즉시 갱신하되, DB 반영은 일정 텀(디바운스/주기적 flush, 약 1~5분 또는 화면을 벗어날 때)을 두고 일괄 저장 — 제스처 프레임마다 DB에 쓰지 않아 불필요한 쓰기 부하를 줄임. 화면을 벗어날 때는 대기 중인 변경분을 즉시 flush해 유실 방지 | 배치 변경 | (지연된) DB 반영 |
| F8.6 | 명시적 저장 버튼 | 상단 "저장" 버튼으로 대기 중인 변경분을 즉시 flush하고 다꾸 스페이스를 확정, 캘린더(또는 날짜별 다꾸 목록)로 복귀. **저장하지 않은 변경 사항이 없으면(스티커/텍스트를 하나도 배치하지 않았거나, 이미 저장된 뒤 아무것도 안 바꿨으면) 버튼은 비활성화** — 변경 없는 저장은 의미가 없고, 빈 페이지가 캘린더에 점으로 찍히는 것도 막아야 함. 실제로 뭔가 배치해야만 버튼이 활성화되고, 그때 비로소 `DecoEntry`가 생성됨 | 저장 액션 | 화면 전환 + 저장 완료 피드백(스낵바 등) |
| F8.7 | 텍스트 추가 | "텍스트 추가" 버튼으로 짧은 문구를 캔버스에 배치 (예: 날짜 코멘트, 한 줄 감상). 텍스트 내용은 탭하여 언제든 재편집 가능, 색상은 T103 테마 팔레트 중 선택 | 텍스트 입력 | 캔버스에 배치된 텍스트 |

### 데이터 모델

```dart
class DecoEntry {
  String id;          // UUID
  DateTime date;       // 시간 제거된 날짜. 하루에 여러 개 허용(UNIQUE 아님) — 여러 장의 다꾸를 만들 수 있음
  DateTime createdAt;  // 같은 날짜 내 정렬 기준(작성 순서)
  DateTime updatedAt;
}

enum DecoPlacementType { sticker, text }

class DecoPlacement {
  String id;             // UUID
  String entryId;        // FK -> DecoEntry
  DecoPlacementType type; // sticker 또는 text — 아래 필드 중 하나만 채워짐
  String? itemId;         // type=sticker일 때: FK -> CollectionItem (스티커로 쓰이는 도감 아이템)
  String? text;           // type=text일 때: 텍스트 내용
  String? textColorHex;   // type=text일 때: 글자색 (T103 팔레트에서 선택)
  double x;               // 캔버스 내 위치, 0.0~1.0 정규화 좌표 (캔버스 크기 변화에 안전)
  double y;
  double scale;
  double rotation;        // radians
  int zIndex;             // 겹침 순서
}
```

### 기술 요구사항
- 정규화(0.0~1.0) 좌표로 저장해 캔버스 위젯 크기가 달라져도(기기별 화면 크기 차이) 배치가 깨지지 않도록 함
- `type='sticker'`일 때 `item_id`는 필수(`text`/`text_color_hex`는 NULL), `type='text'`일 때 `text`는 필수(`item_id`는 NULL) — DB에 `CHECK` 제약으로 강제
- `item_id`가 참조하는 도감 아이템이 삭제되면 해당 배치도 함께 삭제(FK `ON DELETE CASCADE`) — 원본이 없는 스티커는 존재 의미가 없음. 텍스트 배치는 이 제약과 무관
- 제스처 로직은 T203 `sticker_canvas_screen.dart`의 `onScale*` 패턴을 그대로 재사용하되, 데모용 이모지 대신 `type`에 따라 실제 도감 아이템 썸네일(`Image.file`) 또는 `Text` 위젯을 렌더링하도록 교체

### UI 레이아웃
- 상단 AppBar: 날짜 표시(예: "2026년 7월 11일"), 뒤로가기, "저장"(F8.6) 버튼, "이미지로 저장"(F9) 액션
- 본문: 정사각형 또는 화면 비율 캔버스, 배치된 스티커들
- 하단: "스티커 추가" 버튼(도감 아이템 선택 바텀시트, T303 `folder_picker.dart` 패턴과 유사한 시트 UI) + "텍스트 추가" 버튼(텍스트 입력 다이얼로그 → 캔버스 중앙에 배치)

## Feature 9: 다꾸 결과물 저장 (Deco Export)

| 항목 | 내용 |
|------|------|
| **기능 ID** | F9-DECO-EXPORT |
| **목적** | 완성한 다꾸 페이지를 이미지로 남겨 앱 밖에서도 보관/공유 가능하게 함 |

### 세부 기능

| ID | 기능명 | 설명 | 입력 | 출력 |
|----|--------|------|------|------|
| F9.1 | 캔버스 캡처 | 현재 캔버스 화면을 PNG로 캡처 (T202의 `RepaintBoundary.toImage()` 기법 재사용) | 저장 액션 | PNG 이미지 |
| F9.2 | 로컬 저장 | 캡처한 이미지를 앱 저장소에 보관(`images/deco/{entryId}.png` — 하루 여러 장을 구분해야 하므로 날짜가 아닌 엔트리 id로 파일명 지정), 캘린더 썸네일(F7.2)에도 활용 | PNG | 저장된 파일 |
| F9.3 | 공유 | 시스템 공유 시트로 캡처 이미지를 외부(카톡/갤러리 등)로 내보내기 | 공유 액션 | 시스템 공유 시트 |

### 기술 요구사항

| 용도 | 패키지 |
|------|--------|
| 화면 캡처 | 기존 `dart:ui`(T202에서 이미 사용 중, 추가 패키지 불필요) |
| 공유 | `share_plus` (F5-STORAGE 기술 스택에 이미 계획되어 있었으나 아직 미설치) |

## 화면 구성 (Screen Flow)

```
[홈 - 달력 탭] (신규, 도감/폴더 옆 세 번째 하단 탭)
  ├── 월별 캘린더 그리드
  ├── 날짜별 다꾸 썸네일 미리보기 + 개수 배지
  └── 날짜 탭
        ├── 기록 없음 → 새 다꾸 스페이스로 바로 진입
        └── 기록 있음 → 그날의 다꾸 목록 시트 (항목 탭 = 열기, "+ 새 다꾸 추가" = 새로 만들기)

[다꾸 스페이스] (날짜별, 하루 여러 장 가능, 모든 날짜 접근 가능)
  ├── 캔버스 (배치된 스티커/텍스트들)
  ├── 스티커 추가 → 도감 아이템 선택 시트
  ├── 텍스트 추가 → 텍스트 입력 다이얼로그 (F8.7)
  ├── 스티커/텍스트 드래그/확대/회전/삭제
  ├── 배치 변경 시 자동 저장 (F8.5)
  ├── "저장" → 저장 완료 피드백 + 캘린더/다꾸 목록으로 복귀 (F8.6)
  └── "이미지로 저장" → PNG 캡처 → 로컬 저장 + 공유 시트 (F9)
```
